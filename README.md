# 魔法森林大冒险

## 一、项目概述
《魔法森林大冒险》是一个基于 HTML5、CSS3 与 JavaScript 开发的简化冒险类网页小游戏。  
游戏围绕角色控制与关卡挑战展开，玩家需要通过移动、跳跃与技能释放，在复杂地形中生存并完成目标。

本项目采用 **Spec Coding + Vibe Coding** 的开发模式：  
首先通过结构化策划明确系统设计与核心玩法，再在此基础上进行交互体验与细节优化，形成完整的游戏实现流程。

## 三、核心功能设计

### 1. 角色系统
- 实现角色的左右移动与跳跃控制  
- 设计生命值与魔法值双资源系统  
- 支持受伤判定与死亡机制  

### 2. 魔法系统
- 火球术：用于攻击敌人  
- 冰锥术：用于控制与限制敌人移动  
- 技能消耗魔法值，并具备一定释放策略  

### 3. 敌人系统
- 巡逻型敌人：按固定路径移动  
- 陷阱型敌人：结合地形产生威胁  
- 实现基础 AI 行为逻辑  

### 4. 关卡系统
- 共设计 5 个主题关卡  
- 难度随关卡逐步提升  
- 包含不同地形与障碍组合  

### 5. 互动机制
- 冰块跳板：提供额外弹跳能力  
- 地刺陷阱：对角色造成伤害  

### 6. 收集系统
- 设置水晶为核心收集目标  
- 影响关卡通关条件

## 四、运行说明

### 1. 环境要求
- 支持 HTML5 Canvas 的现代浏览器（Chrome / Edge / Firefox）  
- 无需安装额外依赖  

### 2. 运行步骤
1. 克隆或下载项目代码  
2. 打开项目目录  
3. 直接双击以下文件之一：
   - index.html  
   - 魔法森林大冒险 - 完整独立运行版.html  

---

## 五、操作说明

- ← / → ：角色左右移动  
- ↑ / 空格键：跳跃  
- X 键：释放火球术  
- C 键：释放冰锥术  

### 游戏规则
- 通关条件：完成 5 个关卡并收集足够数量的水晶  
- 失败条件：角色生命值降至 0

## 七、开发流程（AI 辅助）

### 1. Spec Coding 阶段
- 分析冒险类小游戏的通用玩法模式  
- 抽象核心循环与交互逻辑  
- 使用 Coze 生成初步游戏策划  
- 使用 Gemini、豆包优化提示词（Prompt）  
- 输出可执行开发规范  

### 2. Vibe Coding 阶段
- 基于策划案生成初始游戏 Demo  
- 优化用户引导与交互提示  
- 调整关卡难度曲线  
- 修复关键逻辑问题与体验问题  

---

## 八、设计思路

### 1. 核心设计理念
- 采用“水晶收集 + 关卡通关”的双重目标机制  
- 技能设计体现差异化（攻击与控制）  
- 敌人类型区分增强策略性  

### 2. 关卡设计思路
- 难度逐步递进  
- 不同关卡主题区分  
- 平台结构与陷阱组合设计  

### 3. 技术实现方案
- 面向对象编程进行模块划分  
- 基于 Canvas 的实时渲染  
- 实现轻量级物理碰撞检测机制
